# Week 6 - Model Pipeline Packaging

## Objective

The objective of this stage is to convert the trained XGBoost fraud
detection model into a reusable inference pipeline that can be used by
the API and dashboard in later stages.

## Artifacts Used

- `models/final_fraud_model.pkl`
- `models/best_threshold.pkl`
- `models/feature_list.pkl`

## Pipeline Logic

1. Load trained model, threshold, and feature list.
2. Validate incoming transaction features.
3. Align feature order with the training feature list.
4. Replace missing or infinite values if needed.
5. Generate fraud probability using the XGBoost model.
6. Apply the selected threshold of `0.73`.
7. Return fraud probability, prediction, prediction label, and threshold.

## Notes

The current inference pipeline expects model-ready features. In a
real-world fraud detection system, these features would typically be
generated by a feature engineering layer or feature store before being
sent to the model.

This stage prepares the trained model for a simulated real-time fraud
detection system. It is not a full production real-time deployment yet.

## Source Code Scope

The active Week 6 source files are:

- `src/validate_artifacts.py`
